# Lesson 1.4 — OpenFOAM Case Structure and File Layout

## Concept

Every OpenFOAM simulation is a directory. The directory layout is **fixed by convention** — OpenFOAM finds your files by looking in specific locations. Understanding this structure is the single most important practical skill in OpenFOAM.

### The Three Pillars

```
myCase/
├── 0/          ← WHAT state the fluid starts in
├── constant/   ← WHAT the physics and geometry are
└── system/     ← HOW to run the simulation
```

---

## The `0/` Directory — Initial & Boundary Conditions

Contains one file per field variable. At time `t = 0`, these files define:
- The **initial condition** (value everywhere in the domain)
- The **boundary condition** (value or behavior at each patch/wall)

Example `0/U` (velocity field):
```cpp
FoamFile
{
    version     2.0;
    format      ascii;
    class       volVectorField;    // U is a vector (3 components)
    object      U;
}
// * * * * * * * * * * //

dimensions      [0 1 -1 0 0 0 0];  // SI units: m/s

internalField   uniform (0 0 0);   // Initially at rest everywhere

boundaryField
{
    movingWall
    {
        type    fixedValue;
        value   uniform (1 0 0);   // Wall moves at 1 m/s in x
    }
    fixedWalls
    {
        type    noSlip;            // No-slip: U = 0 at wall
    }
}
```

### Dimensions Notation `[kg m s K mol A cd]`

```
[0 1 -1 0 0 0 0]
 ↑  ↑  ↑
 kg m  s⁻¹  → m/s (velocity)

[0 2 -2 0 0 0 0]  → m²/s²  (pressure in OpenFOAM = p/ρ)
[0 2 -1 0 0 0 0]  → m²/s   (kinematic viscosity ν)
```

OpenFOAM stores pressure as **kinematic pressure** `p/ρ` (units: m²/s²), not Pa. This surprises everyone at first.

## The `constant/` Directory — Geometry and Properties

```
constant/
├── polyMesh/           ← The mesh (generated by blockMesh or snappyHexMesh)
│   ├── points          ← Coordinates of all mesh vertices
│   ├── faces           ← Connectivity: which points form each face
│   ├── cells           ← Which faces form each cell
│   ├── boundary        ← Patch names and types
│   └── owner / neighbour
└── transportProperties ← Physical properties (viscosity, etc.)
```

Example `constant/transportProperties`:
```cpp
FoamFile { ... }

transportModel  Newtonian;

nu              0.01;   // Kinematic viscosity [m²/s]
                        // Water: ~1e-6, Air: ~1.5e-5
```

For turbulent cases, you'll also have:
```
constant/turbulenceProperties
```

## The `system/` Directory — Solver Control

### `system/controlDict` — Simulation Control

```cpp
FoamFile { ... }

application     icoFoam;      // Which solver to run

startFrom       startTime;    // Start from t=0
startTime       0;

stopAt          endTime;
endTime         0.5;          // Stop at t = 0.5 seconds

deltaT          0.005;        // Timestep size

writeControl    timeStep;     // Write output every N steps
writeInterval   20;

purgeWrite      0;            // Keep all time directories (0 = keep all)

writeFormat     ascii;        // or binary (faster, larger)
writePrecision  6;
```

### `system/fvSchemes` — Numerical Discretization

```cpp
FoamFile { ... }

ddtSchemes
{
    default         Euler;         // 1st order time derivative
}

gradSchemes
{
    default         Gauss linear;  // Linear interpolation gradient
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwind grad(U);  // Upwind for stability
}

laplacianSchemes
{
    default         Gauss linear corrected;
}

interpolationSchemes
{
    default         linear;
}

snGradSchemes
{
    default         corrected;
}
```

### `system/fvSolution` — Linear Solvers

```cpp
FoamFile { ... }

solvers
{
    p                              // Pressure equation
    {
        solver          PCG;       // Preconditioned Conjugate Gradient
        preconditioner  DIC;
        tolerance       1e-06;
        relTol          0.05;
    }

    U                              // Velocity equation
    {
        solver          smoothSolver;
        smoother        symGaussSeidel;
        tolerance       1e-05;
        relTol          0;
    }
}

PISO                               // Pressure-Implicit Split-Operator
{
    nCorrectors     2;
    nNonOrthogonalCorrectors 0;
}
```

## Time Directories

As the simulation runs, OpenFOAM writes output to numbered directories:

```
myCase/
├── 0/          ← initial conditions
├── 0.005/      ← state at t = 0.005 s
├── 0.01/
├── ...
├── 0.5/        ← final state
├── constant/
└── system/
```

Each time directory contains the same fields as `0/` but with the evolved values.

## Key Takeaways

- `0/` = initial and boundary conditions; one file per field variable.
- `constant/` = mesh (polyMesh/) and physical properties.
- `system/` = solver settings: `controlDict` (time), `fvSchemes` (math), `fvSolution` (linear algebra).
- OpenFOAM stores pressure as kinematic pressure `p/ρ` in units of m²/s².
- Dimensions are encoded as `[kg m s K mol A cd]` exponents — always check them.
- Time directories are automatically created as the simulation writes output.
