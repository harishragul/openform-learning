# Lesson 3.1 — Mesh Concepts: Cells, Faces, Nodes, Zones

## Concept

The mesh is the foundation of every CFD simulation. A poor mesh produces wrong answers even with perfect boundary conditions and solver settings. Understanding mesh anatomy is non-negotiable.

### What is a Mesh?

A mesh (or grid) divides the continuous fluid domain into a finite number of small control volumes called **cells**. The Navier-Stokes equations are integrated over each cell — this is the Finite Volume Method.

```
Continuous domain          Discretized mesh (2D)
                    →
 ───────────────          ┌──┬──┬──┬──┬──┐
│               │         ├──┼──┼──┼──┼──┤
│  (fluid)      │         ├──┼──┼──┼──┼──┤
│               │         ├──┼──┼──┼──┼──┤
 ───────────────          └──┴──┴──┴──┴──┘
```

### Mesh Anatomy

| Element | Description | OpenFOAM file |
|---------|-------------|---------------|
| **Points** (nodes) | Vertices of the mesh | `constant/polyMesh/points` |
| **Faces** | Flat polygons connecting points | `constant/polyMesh/faces` |
| **Cells** | Control volumes (bounded by faces) | `constant/polyMesh/cells` |
| **Patches** | Named groups of boundary faces | `constant/polyMesh/boundary` |
| **Zones** | Named groups of cells or faces | Used for sources, refinement |

### Internal vs. Boundary Faces

- **Internal face**: shared between two cells (owner and neighbour)
- **Boundary face**: belongs to only one cell; the other side is a boundary patch

```
cell A │ cell B         ← internal face (between A and B)
───────┤───────
                        (A is owner, B is neighbour)

cell C │ [WALL]         ← boundary face (C is owner, no neighbour)
───────┘
```

### Cell Types

OpenFOAM's polyMesh format supports any convex polyhedron:

| Cell type | Faces | Use case |
|-----------|-------|---------|
| Hexahedron (hex) | 6 | Structured grids, boundary layers |
| Tetrahedron (tet) | 4 | Complex geometry (from CAD) |
| Prism (wedge) | 5 | Axisymmetric cases |
| Pyramid | 5 | Transition regions |
| Polyhedron | N | snappyHexMesh cut cells |

**Hexahedral cells are best** for accuracy and convergence. Always prefer hex where possible.

## Mesh Quality Metrics

`checkMesh` reports these metrics — learn what each means:

### 1. Non-orthogonality
Angle between the vector connecting two cell centers and the face normal.
- **0°** = perfect orthogonal mesh
- **< 70°** = acceptable
- **> 85°** = severe; solver may diverge without non-orthogonal correctors

```
cell A ●─────────●  ← face normal points straight between centers: 0°
       |    face |
cell B ●─────────●

cell A ●                  ← non-orthogonal: vector skewed from face normal
        \   face
cell B   ●────────●
```

### 2. Skewness
How far the intersection of the face normal with the face deviates from the face center.
- **< 0.85** = acceptable
- **> 4** = highly skewed; numerical errors accumulate

### 3. Max aspect ratio
Ratio of longest to shortest edge of a cell.
- For boundary layers: high aspect ratio (e.g., 100:1) is normal and intentional
- For interior cells: keep below 20 for best accuracy

### 4. Minimum cell volume
Very small cells = very small timestep (CFL condition)

## Structured vs. Unstructured Meshes

| | Structured | Unstructured |
|--|-----------|--------------|
| Cell arrangement | Ordered grid (i,j,k indices) | No regular pattern |
| Cell type | Always hexahedral | Tet, hex, mixed |
| Generated by | `blockMesh` | `snappyHexMesh`, Gmsh, others |
| Quality | Better (lower non-orthogonality) | Variable |
| Complex geometry | Difficult | Easy |
| Typical use | Channel flows, simple shapes | Aircraft, engine, biological |

## Key Takeaways

- The mesh divides the domain into cells; equations are solved per-cell.
- Key elements: points → faces → cells → patches.
- Hexahedral cells give best accuracy; use them where possible.
- Non-orthogonality < 70° and skewness < 4 are the critical quality thresholds.
- `checkMesh` is mandatory after every mesh generation step.
- Structured meshes (blockMesh) are better quality; unstructured meshes handle complex geometry.
